## 1 属性选择器

* 场景1:禁用按钮，无小手。
* 以前：
```
button {
    cursor: pointer;
}
.btn_disabled{
    cursor: default;
}

<button>按钮</button>
<button>按钮</button>
<button disabled="disabled" class='btn_disabled'>按钮</button>
<button disabled="disabled" class='btn_disabled'>按钮</button>
```

* 现在：这就C3的属性选择器；
```
button[disabled] {
    cursor: default;
}

button {
    cursor: pointer;
}
<button:1>按钮</button>
<button>按钮</button>
<button disabled="disabled">按钮</button>
<button disabled="disabled">按钮</button>
```

* 引入 属性选择器：

![](./img/6.jpg)

* 注意：
* 1.E[attr]无空格，attr无引号。E可以为标签、类、ID都可以；
* 2.类选择器、属性选择器、伪类选择器，权重为 10，权重不神奇：越低的话，涉及的范围越大，越高，就是小范围的要求；
```

* 场景3：图标不一样，都是一类的标签选择出来
* 以前：
```
  <div class="icon icon1">图标1</div>
  <div class="icon icon2">图标2</div>
  <div class="icon icon3">图标3</div>

  .icon{
    color:#222;
  }
```

* 现在：^：以后面的值开头的属性
```
  div[class^="icon"] {
    color: red;
  }
```

* 同样：以什么什么结尾的属性
```
div[class$="icon"] {
 color: green;
  }
```

* 任意位置出现的：
```
  div[class*="icon"] {
    color: blue;
  }
```

* 总结：
* 1.不神奇：E[attr]无空格，attr无引号。E可以为标签、类、ID都可以；
* 2.属性不要引号，后面的值需要引号，写的方式和标签内部属性的方式一样；
* 3.类选择器、属性选择器、伪类选择器，权重为 10，权重不神奇：越低的话，涉及的范围越大，越高，就是小范围的要求；范围的一样的情况下，就是按照先是综合生效，然后是按照最后的样式生效；


## 2 结构伪类选择器

* 结构、伪类
* 找朋友：找到就是有样式，找不到就没有。

### 2.1 first-child第一个，最后一个
* 场景：就比如说一个ul下一堆li中，我要第一个li，或者最后一个li
* 以前：
```
<li>1</li>
<li>2</li>
<li class='li_3'>3</li>
<li>4</li>

.li_3{

}
```

* 现在：
```
ul li:first-child {
    background-color: pink;
}

ul li:last-child {
    background-color: deeppink;
}


ul li:nth-child(8) {
    background-color: lightpink;
}
```

### 2.2 nth-child(n) 
* 原理：先找到第几个孩子，再看是不是前面的标签，是就命中，不是就没有。

* 1.明确的数字：写的哪个就是哪个；也可以写0，就是命中不到而已。
* 2.关键字
```
ul li:nth-child(even) {
    background-color: pink;
}

ul li:nth-child(odd) {
    background-color: hotpink;
}
```

* 3.公式：n 从0开始算。如果我们自己直接写数字的话，写0是选择不到的。它为什么会从0开始？
```
ul li:nth-child(n) {} 

ul li:nth-child(2n) { 0 2 4 6
    background-color: red;
} 

ul li:nth-child(2n+1) { 1 3 4 6 
 background-color: red;
} 【如果n从1开始，那么直接命中就是3】


ul li:nth-child(5n) { 0 5 10
    background-color: pink;
} 

ul li:nth-child(n+5) { 5 6 7 
    background-color: pink;
} 

ul li:nth-child(-n+5) {  5 4 3 2 1
    background-color: pink;
}【选择前面五个】
```

* 常用的业务场景：类似日期排布，周六日变色：
```
ul li:nth-child(7n+1);周六
ul li:nth-child(7n);周日
```

* 【总结】
* 0.E可以为任何：标签、类、ID
* 1.该结构伪类，常用于就是一个父亲下面的子标签都是一样的标签，我想快速的找第几个。ul li:last-child
* 2.局限性：要理解结构化（css的书写方式，结构，HTML代码父亲下面都是一样的子标签）。更好的写代码；
* 3.不建议的CSS写法，也是对结构的深层理解：默认是找上一层的一个结构，再从该结构下往下找。不建议这种没有结构的书写方式；

</div>
<div>

```

* 5.结构伪类的适用场景：注意上面说的，HTML父级下面的子标签要一样，CSS要带有结构的写法。

### 2.3 div span:nth-of-type(2)

* 解读：先找孩子：找div下span类型的孩子中，第二个孩子，有这个孩子就命中
```
div span:first-of-type {
            background-color: purple;
        }
        
div span:last-of-type {
    background-color: skyblue;
}
```

### 2.4 区别

* 找孩子的方式不一样，也就是我们写代码的时候，侧重的地方不一样：
* ul li:nth-child(2)：找孩子，找下面孩子第二个孩子是不是li，是就命中；先把第二个孩子位置找到，再确认是不是这个孩子是不是li。
* il li:nth-of-type(2):找孩子，找下面孩子都是li类型的中第二个孩子，先把孩子类型找出，再找第几个。
